# Автомат поиска подстроки

## Идея

Хотим искать паттерн $p$ длины $m$ в тексте. Построим **детерминированный конечный автомат (ДКА)**, который читает текст посимвольно и в каждый момент хранит одно число — **сколько символов паттерна сопоставлено прямо сейчас**.

Это число и есть состояние автомата.

---

## Состояния

- Состояний $m + 1$: от $0$ до $m$.
- Состояние $q$ означает: «наибольший суффикс прочитанного текста, совпадающий с префиксом $p$, имеет длину $q$».
- Состояние $0$ — ничего не сопоставлено.
- Состояние $m$ — паттерн найден целиком.

---

## Функция переходов $\delta(q, c)$

Находясь в состоянии $q$ и читая символ $c$, нужно найти наибольший $k$ такой, что:

$$p[0..k-1] \text{ является суффиксом } (p[0..q-1] + c)$$

Иначе говоря — смотрим на строку $p[0..q-1] + c$ и берём длину её наибольшего префикса, совпадающего с суффиксом.

**Формально:**

$$\delta(q, c) = \max\{k : p[0..k-1] \text{ — суффикс } p[0..q-1]+c\}$$

### Три варианта:

1. `p[q] == c` → $\delta(q, c) = q + 1$ (удачное продолжение)
2. `p[q] != c`, `q == 0` → $\delta(q, c) = 0$ (откатываться некуда)
3. `p[q] != c`, `q > 0` → $\delta(q, c) = \delta(\pi[q-1], c)$ (откат через префикс-функцию)

Пункт 3 — ключевой. Вместо того чтобы наивно перебирать все $k$, мы используем уже вычисленный переход из состояния $\pi[q-1]$. Поэтому строить автомат надо **в порядке возрастания $q$**.

---

## Построение за $O(m \cdot |\Sigma|)$

```cpp
#include <vector>
#include <string>
using namespace std;

vector<int> prefix_function(const string& s) {
    int n = s.size();
    vector<int> pi(n, 0);
    for (int i = 1; i < n; i++) {
        int k = pi[i - 1];
        while (k > 0 && s[k] != s[i]) k = pi[k - 1];
        if (s[k] == s[i]) k++;
        pi[i] = k;
    }
    return pi;
}

// delta[q][c] — переход из состояния q по символу c (c = символ - 'a')
vector<vector<int>> build_automaton(const string& pat, int sigma = 26) {
    int m = pat.size();
    auto pi = prefix_function(pat);
    vector<vector<int>> delta(m + 1, vector<int>(sigma, 0));

    for (int q = 0; q <= m; q++) {
        for (int c = 0; c < sigma; c++) {
            if (q < m && pat[q] - 'a' == c)
                delta[q][c] = q + 1;          // удачное продолжение
            else if (q == 0)
                delta[q][c] = 0;              // некуда откатываться
            else
                delta[q][c] = delta[pi[q-1]][c]; // откат
        }
    }
    return delta;
}
```

> Почему `delta[pi[q-1]][c]` уже известен на шаге $q$? Потому что $\pi[q-1] < q$, и мы обходим состояния по возрастанию.

## Поиск за $O(n)$

```cpp
vector<int> search(const string& text, const string& pat) {
    auto delta = build_automaton(pat);
    int m = pat.size(), q = 0;
    vector<int> occ;
    for (int i = 0; i < (int)text.size(); i++) {
        q = delta[q][text[i] - 'a'];
        if (q == m)
            occ.push_back(i - m + 1);
    }
    return occ;
}
```

Каждый символ — ровно одно обращение к таблице: $O(1)$. Итого $O(n)$.

## Пример: `p = "abab"`, $\Sigma = \{a, b\}$

Сначала считаем $\pi$: `[0, 0, 1, 2]`.

| $q$ | сопоставлено | $\delta(q, a)$ | $\delta(q, b)$ |
|-----|-------------|----------------|----------------|
| 0 | `""` | 1 | 0 |
| 1 | `"a"` | 1 | 2 |
| 2 | `"ab"` | 3 | 0 |
| 3 | `"aba"` | 1 | 4 |
| 4 | `"abab"` ✓ | 3 | 0 |

Разберём `δ(2, b)`: `p[2] = 'a' != 'b'`, откат → `δ[π[1]][b] = δ[0][b] = 0`. ✓  
Разберём `δ(4, a)`: `q = m`, откат → `δ[π[3]][a] = δ[2][a] = 3`. ✓

Трассировка текста `"aababab"`:

```
i:  0  1  2  3  4  5  6
c:  a  a  b  a  b  a  b
q:  1  1  2  3  4  3  4
              ↑        ↑
           i=4: occ=1  i=6: occ=3
```

## Потоковый ввод

Автомат идеален для потока — символы приходят один за другим, состояние хранится между вызовами:

```cpp
struct PatternMatcher {
    vector<vector<int>> delta;
    int m, q = 0;

    PatternMatcher(const string& pat)
        : delta(build_automaton(pat)), m(pat.size()) {}

    // возвращает true, если именно на этом символе завершилось вхождение
    bool feed(char c) {
        q = delta[q][c - 'a'];
        return q == m;
    }
};
```

## Связь с КМП

| | КМП (через $\pi$) | Автомат |
|---|---|---|
| Предобработка | $O(m)$ | $O(m \cdot \|\Sigma\|)$ |
| Поиск | $O(n)$ | $O(n)$ |
| Память | $O(m)$ | $O(m \cdot \|\Sigma\|)$ |
| Откат при поиске | есть (цепочка $\pi$) | нет (таблица) |
| Потоковый ввод | неудобно | естественно |
| Много текстов | перестраивать не нужно | автомат строится один раз |

Автомат — это КМП, в котором все откаты **предвычислены** и закешированы в таблице.

## Типичные вопросы

**В.** Почему строим в порядке возрастания $q$?  
**О.** Потому что $\delta(q, c)$ при откате использует $\delta[\pi[q-1]][c]$, а $\pi[q-1] < q$ — значит этот переход уже вычислен.

**В.** Что происходит в состоянии $m$ после нахождения?  
**О.** Автомат не останавливается — из $m$ тоже определены переходы, что позволяет искать **перекрывающиеся** вхождения.

**В.** Чем плоха наивная построение автомата (без $\pi$)?  
**О.** Для каждого $(q, c)$ пришлось бы перебирать все $k$ от $\min(q, m)$ до $0$ — итого $O(m^2 \cdot |\Sigma|)$.
